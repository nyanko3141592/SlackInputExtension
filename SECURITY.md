# セキュリティポリシー

## サポートしているバージョン

最新のリリースのみセキュリティ修正を提供します。

| Version | サポート |
|---|---|
| 最新の `0.x.x` | ✅ |
| それ以前 | ❌ |

## 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、**公開 Issue を立てずに**以下のいずれかの方法で報告してください。

### 推奨: GitHub Security Advisory

1. このリポジトリの [Security タブ](https://github.com/nyanko3141592/SlackInputExtension/security/advisories) を開く
2. 「Report a vulnerability」をクリック
3. 詳細を記入して送信

### 代替手段

GitHub アカウントを持たない場合は、リポジトリオーナーに DM 等でご連絡ください。

## 報告に含めてほしい情報

- 脆弱性の種類（例: XSS, データ漏洩, 認可バイパス）
- 影響範囲（どのバージョン、どの機能）
- 再現手順（できるだけ詳しく）
- 期待される影響（情報漏洩、認証回避等）
- 修正案（あれば）

## 対応プロセス

1. **受領確認**: 報告から 72 時間以内に受領を確認します
2. **トリアージ**: 1 週間以内に重大度を判定します
3. **修正**: 重大度に応じた優先度で対応し、修正後にあなたへ連絡します
4. **公開**: パッチをリリースし、GitHub Security Advisory として公開します（報告者の希望に応じてクレジットします）

## スコープ

### 報告対象 ✅

- API キーの漏洩リスク
- XSS（メッセージリライト経由含む）
- 拡張機能経由でのデータ送信先の改ざん
- chrome.storage への不正書き込み
- Slack DOM 内の機密情報の意図しない送信

### スコープ外 ❌

- Slack 本体の脆弱性（Slack 社へ）
- Google Gemini API 側の脆弱性（Google へ）
- ローカル環境の Chrome 自体の問題
- ユーザー自身が API キーを公開リポジトリに commit してしまった等の運用ミス

## 既知のセキュリティ考慮事項

- API キーは `chrome.storage.sync` に平文で保存されます（Chrome 自体の暗号化に依存）
- リライト対象のテキストは Gemini API に平文で送信されます（HTTPS）
- 拡張機能は `https://*.slack.com/*` のみで動作し、他サイトのデータには触れません

詳細は [PRIVACY.md](./PRIVACY.md) を参照してください。
